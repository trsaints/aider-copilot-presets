# Aider CLI tooling for Copilot-like Experience

This repository provides Aider configuration for providing a CLI experience in similar fashion to GitHub Copilot's tooling, including:

- Sessions centralized: it creates an `$HOME/.aider-sessions/<repo_name>` so your repo may not be polluted by Aider's default behavior;
- Sessions per mode: shell commands are created for starting ask/agent/plan sessions that are named after the first characters of your first prompt:
  - Example: if you prompt `copilot-ask "Explain depency injection in ASP.NET 10"`, everything ongoing on that session is streamed to
    `$HOME/.aider-sessions/<repo_name>/'Explain\ dependency\ injection\ in\ ASP.NET\ 10'.md` session file
  
> [!NOTE]
> If the first prompt exceeds 96 characters, the session file will be named after `'<first_96_characters>...'.md`

---

> [!NOTE]
> By default, this configures integration for OpenRouter's free models via the `openrouter/openrouter/proxy`

## Instructed Constraints

- Ask and Plan modes can not write to files (plan mode may only operate a `plan.md` file generated by the tooling);
- Automatic commits are disabled _for all modes_ (you might not like it if you're as not as distrustful as me);
- Every command to be executed requests user confirmation (you might not like it if you're not as distrustful as me²);
